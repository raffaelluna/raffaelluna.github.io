Detectando Anomalias de Transações Bancárias

Nos últimos tempos tenho me dedicado bastante a me capacitar para vaga de Data Scientist e Machine Learning Engineer. Essa minha busca por uma vaga resultou num projeto muito legal para uma vaga de estágio na One Way Solutions. Era uma oportunidade que consistia basicamente resolver um desafio de Data Science a partir de dados de uma empresa fictícia sem qualquer informação adicional e gerar uma análise rica e um modelo de Machine Learning.

Os notebooks da minha solução para o desafio vocês podem encontrar no repositório do projeto e qualquer dúvida podem me chamar no LinkedIn

O desafio era simular uma situação de um projeto real de uma empresa que trabalha com grandes quantidades de dados. Nesse sentido, foi fornecido um conjunto de dados no formato JSON relativos a transações de cartão de crédito de 9 empresas. Cada arquivo JSON possuia diversos logs de transações, os quais registravam o número do cartão, código de segurança, um timestamp, dia de expiração, um identificador único e o usuário que realizou a transação. Assim, minha tarefa era ler esses arquivos JSON e fazer uma análise completa sobre os dados e produzir um modelo de Machine Learning.

Para resolver este problema, a solução que eu encontrei foi gerar um pipeline de extração de dados que servisse para todas a empresas do conjunto de dados fornecido. Para isso, primeiro era realizado a leitura dos arquivos JSON relativos a uma empresa e armazenados em um dataframe pandas. Em seguida, os tipos de cada variável do dataframe foram corrigidos, algumas novas features foram criadas.

Após isso, partimos para uma primeira análise dos dados. O que se observou foi que foram realizadas 4.100.000 de transações de cartão de crédito por 100.000 usuário, isto é, cada usuário, no período analisado, realizou em média 41 transações. Além disso, os cartões mais utilizados foram o VISA 16 dígitos e o JCB 16 dígitos. Foi possível perceber também que os usuários realizavam mais transações na sexta-feira e menos transações na quarta-feira.

Depois dessa análise inicial, eu decidi construir um novo dataset a partir desse. Esse novo conjunto de dados consistiu em informações sobre cada usuário, como seu total de transações, quantidade de vezes que usou cada bandeira de cartão, velocidade de transações por dia e por hora. Além disso, foram geradas flags para as variáveis de total de transações, velocidade por dia e velocidade por hora, as quais indicam se o usuário é considerado outlier ou não.

Esse novo dataset serviu como subsídio para um modelo não supervisionado de Machine Learning para detecção de anomalias, o One Class Support Vector Machine. Esse algoritmo funciona como um detector de novidade, em que primeiro passamos para ele aprender como são os dados considerados normais e, após isso, ele classifica novos dados como anomalia ou não.

O modelo, por fim, conseguiu encontrar outliers no conjunto de dados, indicados por correlações posivitas entre as flags de suspeição e a resposta do modelo, apesar dessas correlações não serem fortes. Dessa forma, os resultados poderiam ser otimizados caso fossem extraídos mais dados e/ou o modelo fosse aperfeiçoado. Portanto, este modelo seria bastante útil para determinar prioridade de analise para usuários com diversas flags de suspeição.
